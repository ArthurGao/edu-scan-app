# EduScan - Intelligent Learning Assistant

AI-powered educational problem solver backend and mobile application.

## Tech Stack

### Backend
- **Framework**: FastAPI
- **Database**: PostgreSQL + SQLAlchemy 2.0
- **Cache**: Redis
- **AI**: LiteLLM (Claude/GPT/Gemini)
- **Migration**: Alembic

### Mobile
- **Framework**: React Native (Expo)
- **Language**: TypeScript
- **Navigation**: Expo Router

---

## 1. Prerequisites

- **Node.js**: v18+ (Verified v25.4.0)
- **Python**: 3.11+ (Verified v3.13.5)
- **Anaconda/Miniconda**
- **Docker & Docker Compose**
- **API Keys**: Anthropic, OpenAI, or Google (Gemini)

## 2. Backend Setup & Run

### Step 1: Clone and Setup Environment
```bash
cd backend
conda create -n edu-scan-app python=3.13 -y
conda activate edu-scan-app

```bash

```bash

```bash

```bash

## 4. Database Management (Alembic)

This project uses **Alembic** to manage schema changes.

### Standard Workflow
1. **Modify Models**: Update files in `app/models/`.
2. **Generate Migration**:
   ```bash
   alembic revision --autogenerate -m "description of change"
   ```
3. **Review**: Check `alembic/versions/` for correctness.
4. **Apply**: `alembic upgrade head`.

### Manual Database Creation (Docker)
If you need to manually create the database or user in an existing container:
```bash
# Enter container
docker exec -it backend-postgres-1 psql -U postgres

# Run SQL
CREATE DATABASE eduscan;
GRANT ALL PRIVILEGES ON DATABASE eduscan TO postgres;
\q
```

### Common Commands
| Command | Description |
| :--- | :--- |
| `alembic upgrade head` | Update to latest version. |
| `alembic downgrade -1` | Undo last migration. |
| `alembic history` | Show migration versions. |

## 5. RAG Seed Data

EduScan uses a RAG (Retrieval-Augmented Generation) system to improve answer quality and reduce API costs. The seed data pipeline populates the vector database with K12 knowledge and practice problems.

### What gets seeded

| Migration | Table | Content |
|-----------|-------|---------|
| `014_seed_knowledge_base` | `knowledge_base` | 38 K12 concept explanations (math/physics/chemistry) |
| `015_seed_problems` | `scan_records` + `solutions` | ~145 problems generated by Claude |
| `016_seed_problems_gemini` | `scan_records` + `solutions` | ~145 problems generated by Gemini (free) |

> `014` runs automatically with `alembic upgrade head`. `015` and `016` must be **generated once** locally before committing (see below).

### Generate seed problems (run once locally)

**Option A — Claude (~$0.50, higher quality):**
```bash
# Preview (optional)
python -m scripts.generate_seed_problems --dry-run --subjects math --per-topic 2

# Generate → alembic/versions/015_seed_problems.py
python -m scripts.generate_seed_problems --export-migration

git add alembic/versions/015_seed_problems.py
git commit -m "feat: add Claude seed problems for RAG"
```

**Option B — Gemini (free tier, 1500 req/day):**
```bash
# Preview (optional)
python -m scripts.generate_seed_problems_gemini --dry-run --subjects math --per-topic 2

# Generate → alembic/versions/016_seed_problems_gemini.py
python -m scripts.generate_seed_problems_gemini --export-migration

git add alembic/versions/016_seed_problems_gemini.py
git commit -m "feat: add Gemini seed problems for RAG"
```

> Run both options to get ~290 seed problems total. `016` depends on `015` — generate them in order.

Common options (both scripts):
```bash
--subjects math physics chemistry   # which subjects (default: all)
--per-topic 5                       # problems per topic (default: 5)
--export-migration                  # write migration file (default)
--dry-run                           # print only, no files written
```

### Backfill embeddings (run after each deployment)

Migrations insert raw text only. Embeddings are generated separately via OpenAI (~$0.01):

```bash
python -m scripts.backfill_embeddings                    # all tables
python -m scripts.backfill_embeddings --table formulas   # only formulas
python -m scripts.backfill_embeddings --table knowledge_base
python -m scripts.backfill_embeddings --table scan_records
```

### Deployment release command

Set this as the release/post-deploy command on Render, Railway, etc.:

```bash
alembic upgrade head && python -m scripts.backfill_embeddings
```
